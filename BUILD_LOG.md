**Real bugs caught during verification (all fixed, not just noted):**
1. **Windows spawn failure.** `child_process.spawn('claude', ...)` failed with `ENOENT` on this machine because npm installs `claude` as a `.cmd` shim on Windows, and Node refuses to spawn `.cmd` files directly without a shell (a deliberate Node security guard, not a bug in our code). The naive fix, `shell: true`, is explicitly deprecated by Node for this exact call shape because it stops escaping arguments and only concatenates them.
